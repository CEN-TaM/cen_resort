04 · 기획한 서비스

작은 불편에서
서비스 아이디어가 시작됐습니다.

기존 후기 게시판의 불편

PC 기준의 후기 게시판은 자료를 찾아보기도, 후기를 작성하기도 어려웠고 활용도도 다소 떨어져 보였습니다.

  • 모바일 이용 불편
  • 후기 작성 과정 번거로움
  • 휴양소별 정보 탐색 어려움
  • 검색·필터 기능 부족
→

우리가 생각한 개선 방향

모바일 중심으로 바꾸고, 휴양소별로 정보를 확인하며 다양한 조건으로 검색하고 찾아볼 수 있는 구조를 기획했습니다.

  • 모바일 중심 UX
  • 휴양소별 정보 구성
  • 다양한 필터 검색
  • 후기 작성과 공유 개선

AI와 함께 발전한 기획

처음부터 완성된 서비스가 있었던 것은 아닙니다. 각 팀원이 동일한 주제에 대해 각자의 방식으로 프롬프트를 작성하고 AI를 활용하여 프로토타입을 제작했습니다. 서로의 결과물을 공유하고 검토하는 과정을 반복하면서 새로운 아이디어가 더해졌고, 기능은 더욱 구체화되었으며 서비스 방향성도 점차 확장되었습니다.

05 · 실습 과정

AI에게 기획부터 HTML까지
실제로 요청해 보았습니다.

기획하기, 화면정의, 디자인 가이드, HTML 프로토타입 제작까지 하나의 흐름으로 실습했습니다. 결과가 마음에 들지 않으면 다시 요청하고, 다른 AI 도구로 교차 검증했습니다.

아이디어 입력→기획서 생성→화면정의→디자인 가이드→HTML 프로토타입→검토·수정
ChatGPT

Claude

맥락 이해, UI 디자인 개선, 코드 수정에 강점이 있었습니다.

Gemini

IA 구조 정리와 콘텐츠 방향성 검토에 활용했습니다.

Figma

HTML 결과물을 시안으로 확인하고 협업 가능한 형태로 검토했습니다.
